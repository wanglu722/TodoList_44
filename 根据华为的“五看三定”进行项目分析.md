# 华为“五看三定”项目分析
## （一）五看：洞察战略机会点
### 1. 看行业/趋势
- 行业：个人效率工具（待办类）是轻量级生产力工具的细分领域，当前趋势是“极简 + 垂直场景”（比如专注学生/职场新人的轻量化待办）。
- 项目匹配度：符合“轻量级工具”的趋势，代码简单易复用，适合入门级用户。

### 2. 看市场/客户
- 市场：待办工具市场已成熟（如 Todoist），但入门级、无学习成本的工具仍有空白（比如学生/编程初学者需要“无广告、纯本地、代码可自定义”的待办）。
- 目标客户：编程学习者、学生群体（需要简单工具 + 可修改代码的实践场景）。

### 3. 看竞争
- 直接竞争：同类开源待办项目（如 GitHub 上的 Java 轻量级待办）。
- 核心优势：代码极简、无依赖、适合教学场景（比复杂项目更易理解）；
- 核心劣势：功能单一（无云同步、分类等）。

### 4. 看自己
- 优势：掌握核心功能代码，可快速迭代；项目轻量，维护成本低。
- 不足：功能扩展性弱，暂无差异化亮点。

### 5. 看机会
- 机会点：聚焦“编程学习辅助工具”—— 在待办功能基础上，增加“代码注释示例”“功能拓展教程”，成为编程初学者的实践模板项目。

## （二）三定：制定落地策略
### 1. 定战略控制点
- 核心控制点：“极简 + 可拓展” 的代码架构（保持基础功能稳定，预留拓展接口）。

### 2. 定目标
- 短期（1 个月）：完善项目文档，成为 50 + 编程初学者的实践参考项目。
- 长期（3 个月）：新增 1-2 个实用拓展功能（如待办分类），获得 100+Star。

### 3. 定策略
- 功能策略：先补充 “待办分类”“本地文件保存” 等基础拓展功能；
- 推广策略：在编程学习社区（如 CSDN、GitHub 话题）分享 “Java 轻量级待办项目实践” 教程，吸引目标用户。
